# AWS Step Functions - Document Processing Workflow

## Overview

This folder contains the AWS Step Functions state machine definition for processing healthcare documents (hospital bills, insurance claims, and medical records) in the review submission workflow.

The state machine orchestrates a multi-stage AI-powered document processing pipeline that extracts text using AWS Textract and then performs semantic field extraction using either Amazon Bedrock Claude (for bills/claims) or Amazon Comprehend Medical (for medical records).

## State Machine Definition

**File:** `document-processing-state-machine.json`

**Type:** Express Workflow (Synchronous)

**Execution Time:** Typically 5-15 seconds per document

## Workflow States

### 1. ExtractText (Task State)

**Purpose:** Extract text, key-value pairs, and tables from the uploaded document using AWS Textract.

**Service:** AWS Textract `AnalyzeDocument` API

**Features:**
- FORMS analysis - Extracts key-value pairs (e.g., "Patient Name: John Doe")
- TABLES analysis - Extracts tabular data (e.g., itemized bills)

**Input:**
```json
{
  "s3Bucket": "choco-warriors-db-synthetic-data-us",
  "s3Key": "documents/hospitalBills/customer-123/file.pdf",
  "documentType": "hospitalBill",
  "documentId": "doc-uuid-123"
}
```

**Output:**
```json
{
  "textractResult": {
    "raw_text": "Full extracted text...",
    "key_values": {
      "Patient Name": "John Doe",
      "Total Amount": "$1,500"
    },
    "tables": [
      [
        ["Item", "Cost"],
        ["Consultation", "$500"],
        ["Lab Tests", "$1,000"]
      ]
    ]
  }
}
```

**Error Handling:**
- Retries: 3 attempts with exponential backoff (3s, 6s, 12s)
- Catches: `Textract.ProvisionedThroughputExceededException`, Lambda errors
- On failure: Transitions to `ExtractionFailed` state

### 2. RouteByDocumentType (Choice State)

**Purpose:** Route the extracted text to the appropriate NLP analyzer based on document type.

**Routing Logic:**
- `hospitalBill` → `ExtractBillData`
- `insuranceClaim` → `ExtractClaimData`
- `medicalRecord` → `ExtractMedicalData`
- Unknown type → `UnknownDocumentType` (Fail state)

### 3a. ExtractBillData (Task State)

**Purpose:** Extract structured payment information from hospital bills using Amazon Bedrock Claude.

**Service:** Amazon Bedrock (Claude 3 Sonnet)

**Extractor:** `aws/lambda/reviewFunction/extractors/bill_extractor.py`

**Extracted Fields:**
```python
{
  "valid": True,
  "reason": "Valid hospital bill",
  "payment": {
    "billNo": "BILL-2024-001",
    "totalBillAmount": 150000,
    "amountToBePayed": 30000,
    "description": "Cardiac surgery with post-op care"  # Generated by Bedrock
  }
}
```

**Bedrock Prompt Strategy:**
- Analyzes raw text, key-value pairs, and tables
- Extracts bill number, total amount, amount to be paid
- Generates human-readable description of services
- Returns structured JSON

**Error Handling:**
- Retries: 2 attempts with 1.5x backoff
- On failure: Transitions to `ExtractionFailed`

### 3b. ExtractClaimData (Task State)

**Purpose:** Extract insurance claim information using Amazon Bedrock Claude.

**Service:** Amazon Bedrock (Claude 3 Sonnet)

**Extractor:** `aws/lambda/reviewFunction/extractors/claim_extractor.py`

**Extracted Fields:**
```python
{
  "valid": True,
  "reason": "Valid insurance claim",
  "claim": {
    "claimId": "CLM-2024-001",
    "claimAmountApproved": 120000,
    "remainingAmountToBePaid": 30000
  },
  "payment": {
    "totalBillAmount": 150000,
    "amountToBePayed": 30000
  }
}
```

**Bedrock Prompt Strategy:**
- Identifies claim ID and approval amounts
- Calculates remaining patient responsibility
- Cross-references with bill amounts if available

### 3c. ExtractMedicalData (Task State)

**Purpose:** Extract medical entities from patient records using Amazon Comprehend Medical.

**Service:** Amazon Comprehend Medical `DetectEntitiesV2` API

**Extractor:** `aws/lambda/reviewFunction/extractors/medical_extractor.py`

**Extracted Fields:**
```python
{
  "valid": True,
  "reason": "Medical entities extracted",
  "confidence": 0.92,
  "extractedData": {
    "hospitalName": "Apollo Hospital",
    "doctorName": "Dr. Rajesh Kumar",
    "surgeryType": "Cardiac Bypass Surgery",
    "procedureDate": "2024-01-15",
    "diagnosis": "Coronary Artery Disease",
    "medications": ["Aspirin", "Atorvastatin", "Metoprolol"]
  },
  "purposeOfVisit": "Cardiac Bypass Surgery"
}
```

**Comprehend Medical Entity Types:**
- `MEDICAL_CONDITION` → diagnosis
- `MEDICATION` → medications list
- `PROCEDURE_NAME` → surgeryType
- `DATE` → procedureDate
- `NAME` → doctorName (heuristic: "Dr." prefix)

**Confidence Scoring:**
- Average confidence across all detected entities
- Minimum threshold: 0.7 (70%)

## Integration with reviewFunction Lambda

The Step Functions state machine invokes the `reviewFunction` Lambda with different `action` parameters:

| Action | Purpose | Extractor Module |
|--------|---------|------------------|
| `textract_extract` | Extract text via Textract | `textract_utils.py` |
| `extract_bill` | Extract bill payment data | `extractors/bill_extractor.py` |
| `extract_claim` | Extract claim data | `extractors/claim_extractor.py` |
| `extract_medical` | Extract medical entities | `extractors/medical_extractor.py` |

**Lambda Handler Routing:**
```python
# In reviewFunction/lambda_function.py
if action == "textract_extract":
    return textract_utils.extract_text(s3_bucket, s3_key)
elif action == "extract_bill":
    return bill_extractor.extract(raw_text, key_values, tables, document_id)
elif action == "extract_claim":
    return claim_extractor.extract(raw_text, key_values, tables, document_id)
elif action == "extract_medical":
    return medical_extractor.extract(raw_text, key_values, tables, document_id)
```

## Invocation Flow

### From Frontend (reviewApi.ts)

1. **Upload to S3:**
   ```typescript
   POST /reviews/presign
   → Get pre-signed URL
   → PUT to S3 directly
   ```

2. **Trigger Processing:**
   ```typescript
   POST /reviews/process-document
   Body: {
     documentId: "doc-uuid",
     s3Key: "documents/hospitalBills/customer-123/file.pdf",
     documentType: "hospitalBill"
   }
   ```

3. **reviewFunction Lambda:**
   - Invokes Step Functions state machine (Sync Express)
   - Waits for completion (5-15 seconds)
   - Returns extracted data to frontend

4. **Frontend Receives:**
   ```json
   {
     "valid": true,
     "payment": { ... },
     "documentId": "doc-uuid",
     "s3Url": "https://..."
   }
   ```

## Error States

### UnknownDocumentType (Fail State)

**Trigger:** Invalid `documentType` in input

**Error:** `UnknownDocumentType`

**Cause:** `documentType must be one of: hospitalBill, insuranceClaim, medicalRecord`

### ExtractionFailed (Fail State)

**Trigger:** Any unhandled error in Textract or NLP extraction

**Error:** `ExtractionFailed`

**Cause:** `Text extraction or NLP analysis step failed.`

**Common Causes:**
- Textract API throttling
- Invalid S3 object (corrupted file, unsupported format)
- Bedrock/Comprehend Medical API errors
- Lambda timeout (rare with Express workflows)

## Deployment

### Prerequisites

1. **reviewFunction Lambda** must be deployed with:
   - Textract permissions
   - Bedrock permissions (Claude model access)
   - Comprehend Medical permissions
   - S3 read permissions

2. **Environment Variables:**
   ```bash
   S3_BUCKET=choco-warriors-db-synthetic-data-us
   BEDROCK_MODEL_ID=anthropic.claude-3-sonnet-20240229-v1:0
   AWS_REGION=us-east-1
   ```

### Create State Machine

```bash
aws stepfunctions create-state-machine \
  --name DocumentProcessingWorkflowUS \
  --definition file://document-processing-state-machine.json \
  --role-arn arn:aws:iam::ACCOUNT_ID:role/StepFunctionsExecutionRole \
  --type EXPRESS
```

### Update State Machine

```bash
aws stepfunctions update-state-machine \
  --state-machine-arn arn:aws:states:REGION:ACCOUNT_ID:stateMachine:DocumentProcessingWorkflowUS \
  --definition file://document-processing-state-machine.json
```

## Cost Optimization

### Textract
- **Cost:** $1.50 per 1,000 pages (FORMS + TABLES)
- **Optimization:** Use FORMS only for simple documents

### Bedrock Claude
- **Cost:** ~$0.003 per 1K input tokens, ~$0.015 per 1K output tokens
- **Optimization:** Minimize prompt size, use Claude 3 Haiku for simpler extractions

### Comprehend Medical
- **Cost:** $0.01 per 100 characters
- **Optimization:** Truncate very long medical records to first 10,000 characters

### Step Functions Express
- **Cost:** $1.00 per 1M requests + $0.00001667 per GB-second
- **Optimization:** Already using Express (cheaper than Standard)

## Troubleshooting

### Issue: Textract returns empty text

**Cause:** Scanned image quality too low, or unsupported file format

**Solution:** 
- Ensure images are at least 150 DPI
- Convert to PDF if possible
- Check file is not password-protected

### Issue: Bedrock extraction returns invalid JSON

**Cause:** Claude occasionally returns malformed JSON

**Solution:**
- Retry logic already in place (2 attempts)
- Consider adding JSON validation in extractor
- Use Claude 3.5 Sonnet for better JSON adherence

### Issue: Comprehend Medical low confidence

**Cause:** Medical terminology not recognized, or text is too noisy

**Solution:**
- Pre-process text to remove headers/footers
- Focus on clinical notes sections only
- Consider custom entity recognition model

### Issue: Step Function timeout

**Cause:** Large documents (>100 pages) take too long

**Solution:**
- Increase Lambda timeout (currently 60s)
- Consider async processing for very large documents
- Split large PDFs into smaller chunks
